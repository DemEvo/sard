# T-Spec: Обновление 1.5 (Desktop XL UI & Smart Start Engine)

## 1. Общие сведения (Front Matter)

* **Название проекта:** AI T-Spec Generator (SARD)
* **Версия спецификации:** 1.5 (Update Sprint)
* **Затрагиваемые модули:** Frontend (`index.html`, `style.css`, `app.js`), Backend (`app.py`).
* **Цель:** Адаптация интерфейса под сверхширокие экраны (24" XL Desktop), улучшение эргономики ввода текста и реализация недостающей логики первичной диспетчеризации брифа (Smart Start).

## 2. Архитектурные ограничения (Constraints)

* ❌ **ЗАПРЕЩЕНО:** Использование медиа-запросов (`@media`) для адаптации под мобильные устройства (смартфоны/планшеты). Приложение строго десктопное.
* ✅ **ТРЕБОВАНИЕ (UI):** Минимальное целевое разрешение экрана — 1920x1080. Максимальное использование горизонтального пространства.
* ✅ **ТРЕБОВАНИЕ (LLM Routing):** Жесткое разделение потоков в `app.py`. Младшая модель (`gemini-2.5-flash`) для парсинга, старшая (`gemini-3.1-pro-preview`) для дебатов.

## 3. Требования к интерфейсу (UI/UX Desktop XL)

**FR-UI1: Просторная зона ввода (Editor Space)**

* **Проблема:** Текущее поле ввода — "узкая бойница", неудобная для вставки больших брифов.
* **Решение:** В `index.html` и `style.css` поле ввода сообщения должно быть заменено на полноразмерный `<textarea>`.
* **Параметры:** Минимальная высота — 5-7 строк текста (`min-height: 120px`). Ширина — 100% доступной области внутри аккордеона. Поле должно поддерживать изменение размера по вертикали (resize: vertical) и перенос строк по `Shift+Enter` (отправка по `Enter`).

**FR-UI2: Контекстные подсказки (Onboarding Hints)**

* **Проблема:** Пользователь видит пустой чат и не понимает, что от него ждет ИИ на конкретном этапе.
* **Решение:** В `app.js` при рендеринге содержимого активного шага, над зоной чата должен динамически или статически выводиться блок-подсказка.
* **Пример контента для шага 1:** *"Опишите бизнес-идею, целевую аудиторию и модель монетизации. ИИ-агенты (Product Manager и Инвестор) помогут выявить слепые зоны."*

**FR-UI3: Компоновка Desktop XL**

* Контейнер приложения (`.app-container`) должен расшириться. Максимальная ширина (`max-width`) может быть увеличена до `1600px` или `1800px`.
* Боковая панель с файлами (`#file-uploader`) и основная зона аккордеона (`#accordion-container`) должны быть расположены в сетке (CSS Grid или Flex), где аккордеон занимает 75-80% ширины, давая максимум места для чтения текстов T-Spec.

## 4. Требования к Бэкенду: Диспетчер Smart Start (FR-0)

Текущий бэкенд работает линейно и отправляет все запросы в боевую модель. Необходимо перехватывать первое сообщение пользователя и маршрутизировать его через AI-Диспетчер.

**Алгоритм модификации эндпоинта `/api/projects/<id>/chat` в `app.py`:**

1. **Условие перехвата (Interceptor Trigger):**
При получении POST-запроса бэкенд проверяет два условия:
* `project.current_state == 'BUSINESS_TASK_ANALYSIS'`
* История чата для данного проекта абсолютно пуста (нет записей в `ChatHistory` для этого проекта).


2. **Действие системы (Dispatcher Execution):**
Если условия выполнены, бэкенд **не вызывает** стандартный промпт шага 1. Вместо этого он:
* Загружает системный промпт `Prompts/SYS_DISPATCHER_m1.md`.
* Инициализирует модель `gemini-2.5-flash` (обязательно с форматом вывода JSON).
* Отправляет первое сообщение пользователя (бриф) в эту модель.


3. **Обработка ответа (State Pre-filling):**
* Бэкенд парсит полученный JSON. Ожидаемая структура (согласно промпту диспетчера) должна содержать ключи для разных этапов (например, `business_data`, `architecture_data`, `nfr_data`).
* Если модель нашла архитектурные данные (как в твоем примере с Supabase) или НФТ, бэкенд программно создает новые записи в таблице `approved_blocks` для соответствующих `state_id` (`PROJECT_CONTEXT_MODELING`, `NFR_FORMALIZATION`) и устанавливает им `status = 'draft'`.


4. **Ответ клиенту (Client Response):**
* После распределения данных, бэкенд сохраняет исходное сообщение пользователя в БД и возвращает фронтенду системное сообщение от Фасилитатора:
* *"Ваш бриф успешно проанализирован. Технические и архитектурные данные сохранены в черновики соответствующих разделов. Давайте сосредоточимся на бизнес-целях. [Далее задать вопрос по бизнесу]"*.
